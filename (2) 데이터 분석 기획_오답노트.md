## 2과목 [데이터 분석 기획] 오답노트

### 1장 데이터 분석 기획의 이해
✏️ KDD 분석 방법론에서 변수 생성, 차원 축소 = **데이터 변환**

✏️ 데이터 탐색은 **데이터 이해 단계**의 과정

✏️ 모델링 단계 : 모델링 **기법 선택** → 테스트 **계획 설계** → 모델 **평가** 

✏️ 프로젝트 위험계획 수립 : **회피, 전이, 완화, 수용**

✏️ 빅데이터 분석 방법론 중 1단계인 분석기획

1. **범위** 설정

2. 프로젝트 **정의**

3. 수행계획 **수립**

4. 분석 **위험** 식별

✏️ **하향식 접근법** : 빠짐없이 문제도출, 해결의 가치에 중점, 모델 캔버스는 탐색 도구로 유스케이스 + 새로운 이슈 탐색

✏️ 하향식 접근법의 단계 : 문제 **탐색** → 문제 **정의** → 해결방안 탐색 → 타당성 검토

✏️ 분석 기회 발굴의 범위 확장
- 시장니즈 탐색 (**고객, 채널, 영향자들**)
- 경쟁자 확대 (**대체재, 경쟁자, 신규 진입자**)

✏️ **채널 (Channel)**
- 고객에게 밸류 프로포지션 전달
- 구매 고객에 대한 A/S 서비스
- 상품/서비스에 대한 고객의 이해를 높여줌

✏️ **상향식** : 프로토타입 이용 모델설계, 인사이트 도출 후 **반복적인** 시행착오로 수정, 문제 도출

✏️ 분석과제 발굴의 상향식 접근

1. 프로세스 **분류**

2. 프로세스 **흐름 분석**

3. 분석 요건 **식별**

4. 분석 요건 **정의**

✏️ **정확도 (Accuracy)** : 분석의 활용적 측면 / **정밀도 (Precision)** : 안정성 측면

### 2장 분석 마스터 플랜

✏️ 투입 비용은 분석 마스터 플랜과 관련 X

✏️ **분석 성숙도** : 분석 역량 진단, 효율성 평가

✏️ **데이터 분석 거버넌스** (**원칙 + 조직 + 프로세스**)
- 품질 관리
- 보안/프라이버시
- 분석 프로세스 표준화

✏️ **데이터 표준화** : 데이터 표준 용어 설정, 명명 규칙 수립, 메타데이터 구축 데이터 사전 구축

✏️ 분석 과제 관리
- KSA ISO 21500:2013 가이드로
- 일정계획 수립 → 수집에 대한 **철저한 통제 + 관리**
- 보고서/시스템에 따라 **관리**에 차이 O

✏️ 풀(Pool)로 관리하는건 확정 안된 분석과제 !
